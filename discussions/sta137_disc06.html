Forecasting Example

Data

The data for this example is in the file disc06data.txt. Read it into R and save it as a numeric vector.

x = read.table('disc06data.txt')
x = x[,1]

Inspect the Data

Let’s take a look at the time series plot, and the sample ACF and PACF plots.

plot.ts(x)

acf(x)

pacf(x)

Comments?

Note that by default, the sample ACF values are plotted starting at lag 0, but the sample PACF values are plotted starting at lag 1. This could be confusing, and might even lead to mistakes if the plots are used as a quick way to choose a candidate models.

If presenting the plots, then it is a good idea to clean them up a little. Let’s delete the lag 0 sample ACF value (why don’t we care about it?), and use the same y-axis scale for the two plots. This can be done using the xlim and ylim arguments, and the actual values depend on the data.

par(mfrow = c(2,1))
acf(x, xlim = c(1,15), ylim = c(-0.5, 0.5))
pacf(x, xlim = c(1,15), ylim = c(-0.5, 0.5))

par(mfrow = c(1,1))

Try Several Models

From the above plots we see that the sample ACF is insignificant after lag 1, and the sample PACF is insignificant after lag 1. The sample ACF doesn’t exactly cut off after lag 3, but oscillates close to 0. It’s not entirely clear whether we should fit an AR, MA or ARMA model. We’ll try several choices and compare.

fitAR1 = arima(x,order=c(1,0,0))
fitAR2 = arima(x,order=c(2,0,0))
fitAR3 = arima(x,order=c(3,0,0))
fitMA1 = arima(x,order=c(0,0,1))
fitMA2 = arima(x,order=c(0,0,2))
fitMA3 = arima(x,order=c(0,0,3))
fitARMA11 = arima(x,order=c(1,0,1))
fitARMA22 = arima(x,order=c(2,0,2))
fitARMA12 = arima(x,order=c(1,0,2))
fitARMA21 = arima(x,order=c(2,0,1))

Choose the Best Model

We now have ten candidate models, and we need to choose the best one among them. We will do it using the AICc criterion, which is commonly used in time series analysis.

I don’t believe R has a function that computes the AICc of an ARIMA model fit to data, but it’s not hard to compute it from the AIC value, which we can get using aic() or from the fitted model. First off, we can always compute it directly from the formula using the known values of \(n\) (sample size) and \(p\) (number of parameters estimated in model, not including \(\sigma\)).

n = length(x); n

fitAR1$aic + 2*2*(2+1)/(n-2-1)

fitAR2$aic + 2*3*(3+1)/(n-3-1)
## [1] 267.7779
fitAR3$aic + 2*4*(4+1)/(n-4-1)
## [1] 269.6107
fitMA1$aic + 2*2*(2+1)/(n-2-1)
## [1] 268.7209
fitMA2$aic + 2*3*(3+1)/(n-3-1)
## [1] 267.7177
fitMA3$aic + 2*4*(4+1)/(n-4-1)
## [1] 269.6249
fitARMA11$aic + 2*3*(3+1)/(n-3-1)
## [1] 268.0681
fitARMA22$aic + 2*5*(5+1)/(n-5-1)
## [1] 271.0702
fitARMA12$aic + 2*4*(4+1)/(n-4-1)

fitARMA21$aic + 2*4*(4+1)/(n-4-1)

To expedite the process, I provided a function aicc(), which will compute the AICc value for an ARIMA object in R (output of functions such as arima(), auto.arima(), and similar ones).

The code for this function is in the file in the R Files section of Resources on Smartsite. First, download the file to your computer. It would be easiest if you save it in the same directory (folder) as the script in which you’re trying to use it. Finally, before using it in your code, include the line source(aicc.R). If the file is saved in a different directory than the one you’re working from, then you need to provide a full path from the current directory to the one with the file.

source('aicc.R')
aicc(fitAR1)
## [1] 266.9207
aicc(fitARMA11)
## [1] 268.0681

Based on AICc, the best model is MA(2). Let’s look at this model closer.

Inspect the Residuals

Let’s look at the coefficients of the model.

mod = fitMA2
mod$coef
##         ma1         ma2   intercept 
##  0.51275241  0.19754647 -0.01795916

If the model is a good fit, then the residuals should resemble white noise.

res = mod$res
ts.plot(res)

acf(res)

pacf(res)

hist(res)

qqnorm(res); qqline(res)

The residuals resemble white noise, so the model seems to be a good fit. If the residuals didn’t look like white noise, then we’d have to consider different models until we find one for which the residuals are close to WN.

Forecasts

Now that we found a model that fits the data reasonably well, let’s use it to make forecasts 5 time points into the future. This can easily be done with the predict() function.

h = 5
predict(mod, n.ahead=h)
## $pred
## Time Series:
## Start = 101 
## End = 105 
## Frequency = 1 
## [1]  0.84042151  0.28628945 -0.01795916 -0.01795916 -0.01795916
## 
## $se
## Time Series:
## Start = 101 
## End = 105 
## Frequency = 1 
## [1] 0.8842544 0.9937206 1.0089570 1.0089570 1.0089570

Note that the output consists of two vectors. These are the forecast values and their standard errors, respectively. We can extract the elements individually.

fcast = predict(mod,n.ahead=h)
f.vals = fcast$pred
f.se = fcast$se

Plotting Forecasts

Finally, it is useful to plot the forecast values to see how they compare to the rest of the sample. This can be done with the points() commands. Make sure to keep track of the time values (X-axis).

plot.ts(x,xlim=c(0,n+h))
points(x=n+1:h, y=f.vals, col='purple',type='b',pch=19)

Let’s add forecast margins of one and two standard errors. I’ll use the function polygon() here as a quick way of doing that.

upper1 = f.vals + f.se
lower1 = f.vals - f.se
upper2 = f.vals + 2 * f.se
lower2 = f.vals - 2 * f.se

plot.ts(x,xlim=c(0,n+h))
polygon(x=c(n+(1:h),n+(h:1)), y=c(upper2,lower2[h:1]), col='lightblue', border=NA)
polygon(x=c(n+(1:h),n+(h:1)), y=c(upper1,lower1[h:1]), col='blue', border=NA)
points(x=n+1:h, y=f.vals, col='darkblue',type='b',pch=19)

Connection with Causality

Recall that a big practical benefit of the causal representation is its usefulness in computing forecast errors. Let’s take a closer look.

First, take a look at all the stuff that an ARIMA object contains.

mod

## arima(x = x, order = c(0, 0, 2))

names(mod)
##  [1] "coef"      "sigma2"    "var.coef"  "mask"      "loglik"   
##  [6] "aic"       "arma"      "residuals" "call"      "series"   
## [11] "code"      "n.cond"    "nobs"      "model"

In particular, we can access the estimated coefficients and noise variance as follows.

mod$coef
##         ma1         ma2   intercept 
##  0.51275241  0.19754647 -0.01795916
sigma.hat = sqrt(mod$sigma2)

Estimates of the coefficents of the causal representations (\(\hat{\psi}_j, j = 1, 2, 3, \cdots\) in lecture notation) can be computed from the estimated model coefficients using ARMAtoMA. Usually the first 20 or so are enough for practical purposes.

psi = ARMAtoMA(ar=NULL, ma=mod$coef[1:2], lag.max=20)
psi
##  [1] 0.5127524 0.1975465 0.0000000 0.0000000 0.0000000 0.0000000 0.0000000
##  [8] 0.0000000 0.0000000 0.0000000 0.0000000 0.0000000 0.0000000 0.0000000
## [15] 0.0000000 0.0000000 0.0000000 0.0000000 0.0000000 0.0000000

Note that in this case, since the model is a moving average, it is its own causal representation and the estimated \(\phi_j\) are equal to the estimated \(\theta_j\). However, the above code can be used to get \(\phi_j\) estimates of any causal ARMA model. One just needs to supply the vectors of AR and MA coefficients, and the maximum value of \(j\) for the \(\hat{\phi}_j\).

We can now apply the formulas from Lecture Notes 13-14 to compute the estimated forecast standard error. \(\hat{\sigma}(h)\) from the \(\hat{\phi}_j\).

First off, \(\hat{\sigma}^2(1) = \hat{\sigma}^2\). For any \(k>1\), we can compute \(\hat{\sigma}^2(k)\) using the following code, and appropriate values of \(k\).

k = 2
(1 + sum(psi[1:(k-1)]^2))*sigma.hat^2
## [1] 0.9874806

To expedite the process, we can use sapply() to apply the above function to all lags \(k = 2, 3, \cdots h\), where \(h\) is our forecast horizon. Don’t forget to take the square root if you’re looking for standard error, not variance.

h=5
my.se = c(1,sapply(X=2:h, FUN = function(k){(1+sum(psi[1:(k-1)]^2))})) * sigma.hat^2 
my.se = sqrt(my.se)

Let’s check how these estimated forecast standard errors compare to the ones computed by predict().

my.se
## [1] 0.8842544 0.9937206 1.0089570 1.0089570 1.0089570
